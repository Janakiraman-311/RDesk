1.  Project Overview
RDesk is a new open-source R package that gives R developers the ability to build commercial-quality, standalone desktop applications — with no dependency on Node.js, Electron, or any external runtime other than R itself.
R is the only major data-science language with no first-party desktop application framework. Python has tkinter (stdlib), Java has JavaFX (JDK), C# has WPF/MAUI (Microsoft first-party), and JavaScript has Electron/Tauri. R has nothing equivalent — Shiny is a web framework, and RInno is a packaging wrapper, not a framework.
RDesk changes that. The architecture is:
A native OS window opened via a lightweight C++ webview launcher (cross-platform: Win/macOS/Linux)
An httpuv WebSocket server running inside R on a random local port
A JSON message protocol bridging R logic ↔ the embedded browser UI
A clean R6-based API that the app developer writes entirely in R
A build tool that produces a self-contained distributable (zip / installer / .app)

VISION
An R developer should be able to write App$new(title="Sales Dashboard") and ship a professional desktop app to a client — with no knowledge of C++, JavaScript build tools, or OS packaging. RDesk owns all of that complexity internally.

2.3  The C++ webview library (the window layer)
The open-source library webview/webview (github.com/webview/webview) provides exactly what we need: a single C++ header that opens a native OS window and embeds a web renderer. It uses:
WebView2 (Chromium-based) on Windows — ships with Windows 10/11
WebKit / WKWebView on macOS — no install needed
WebKitGTK on Linux — available via package manager
The library is ~1,500 lines of C++, MIT licensed, actively maintained, and used by Tauri, Wails, and others. We will wrap it with Rcpp to expose a single R function: rdesk_open_window(url, title, width, height).

KEY INSIGHT
No one has wrapped webview/webview for R with Rcpp and paired it with an httpuv IPC bridge. That is the innovation. Everything else in this stack is proven CRAN-stable code.

3.  Architecture — How RDesk Works
The following describes the complete runtime flow of a running RDesk application.
3.1  Startup sequence
Developer calls app$run() in their R script
RDesk starts an httpuv server on a randomly chosen free port (e.g. 49217)
RDesk calls the Rcpp-wrapped webview launcher: opens a native OS window pointing at http://127.0.0.1:49217
The window loads the app's HTML/CSS/JS UI from the httpuv server
The bundled rdesk.js client library in the HTML establishes a WebSocket connection back to R on the same port
app$on_ready() fires — developer sets up menus, loads data, registers message handlers
The app is live and interactive
3.2  Message protocol (R ↔ UI)
All communication uses JSON messages over WebSocket. The protocol is simple:
// UI → R  (JavaScript sending a message to R)
rdesk.send("get_data", { filter: "active" })

// R → UI  (R responding with data)
app$on_message("get_data", function(msg) {
  app$send("data_result", list(rows = nrow(df), data = df))
})

// UI receiving the response
rdesk.on("data_result", function(payload) {
  renderTable(payload.data)
})

The agent must create src/window.cpp with the following content:
#include <Rcpp.h>
#include "webview.h"
#include <string>
#include <thread>

// [[Rcpp::export]]
void rdesk_open_window(
    std::string url,
    std::string title,
    int width  = 1200,
    int height = 800,
    bool resizable = true
) {
  // Must run on main thread on macOS
  webview::webview w(false, nullptr);
  w.set_title(title);
  w.set_size(width, height,
    resizable ? WEBVIEW_HINT_NONE : WEBVIEW_HINT_FIXED);
  w.navigate(url);
  w.run();
}

// [[Rcpp::export]]
std::string rdesk_platform() {
#if defined(_WIN32)
  return "windows";
#elif defined(__APPLE__)
  return "macos";
#else
  return "linux";
#endif
}

9.  inst/www/rdesk.js — Client Library
This JavaScript file must be included in every RDesk app's HTML. It handles the WebSocket connection to R and exposes the rdesk.send() / rdesk.on() API. The agent must create this file:
(function (global) {
  "use strict";

  let _ws        = null;
  let _handlers  = {};
  let _readyQ    = [];
  let _connected = false;

  function connect(port) {
    _ws = new WebSocket("ws://127.0.0.1:" + port);

    _ws.onopen = function () {
      _connected = true;
      _readyQ.forEach(function (fn) { fn(); });
      _readyQ = [];
    };

    _ws.onmessage = function (evt) {
      try {
        const msg = JSON.parse(evt.data);
        const handlers = _handlers[msg.type] || [];
        handlers.forEach(function (h) { h(msg.payload); });
      } catch (e) {
        console.error("[rdesk] bad message", e);
      }
    };

    _ws.onclose = function () { _connected = false; };
    _ws.onerror = function (e) { console.error("[rdesk] ws error", e); };
  }

  const rdesk = {
    init: function (port) { connect(port); },

    send: function (type, payload) {
      const msg = JSON.stringify({ type: type, payload: payload || {} });
      if (_connected) {
        _ws.send(msg);
      } else {
        _readyQ.push(function () { _ws.send(msg); });
      }
    },

    on: function (type, handler) {
      if (!_handlers[type]) _handlers[type] = [];
      _handlers[type].push(handler);
    },

    off: function (type, handler) {
      if (!_handlers[type]) return;
      _handlers[type] = _handlers[type].filter(function (h) {
        return h !== handler;
      });
    },

    ready: function (fn) {
      if (_connected) { fn(); } else { _readyQ.push(fn); }
    }
  };

  global.rdesk = rdesk;
})(window);

10.  R/App.R — R6 Class Skeleton
The agent must create this file. It is the entire public-facing API. Start with this skeleton and expand in later phases:
#' @title RDesk Application Class
#' @description Create and manage a native desktop window from R.
#' @export
App <- R6::R6Class("App",
  public = list(

    initialize = function(title="RDesk App", width=1200, height=800, icon=NULL) {
      private$.title  <- title
      private$.width  <- width
      private$.height <- height
      private$.icon   <- icon
      private$.handlers <- list()
    },

    on_ready = function(fn) {
      private$.ready_fn <- fn
      invisible(self)
    },

    on_message = function(type, fn) {
      private$.handlers[[type]] <- fn
      invisible(self)
    },

    send = function(type, payload = list()) {
      msg <- jsonlite::toJSON(list(type=type, payload=payload), auto_unbox=TRUE)
      if (!is.null(private$.ws_session)) {
        private$.ws_session$send(msg)
      }
      invisible(self)
    },

    load_ui = function(path) {
      # served by httpuv — just signal the window to navigate
      self$send("__navigate__", list(path = path))
      invisible(self)
    },

    set_menu  = function(items) { invisible(self) },  # Phase 2
    dialog_open = function(...) { NULL },             # Phase 2
    dialog_save = function(...) { NULL },             # Phase 2
    notify    = function(title, msg) { invisible(self) }, # Phase 2
    set_tray_icon = function(path) { invisible(self) },   # Phase 2
    quit = function() { private$.running <- FALSE },

    run = function() {
      private$.running <- TRUE
      port <- private$find_free_port()
      private$start_server(port)
      Sys.sleep(0.3)  # let httpuv bind
      url  <- paste0("http://127.0.0.1:", port)
      # TODO Phase 1: replace with rdesk_open_window() Rcpp call
      message("[RDesk] App running at ", url)
      message("[RDesk] Phase 1 pending: native window not yet opened")
      if (!is.null(private$.ready_fn)) private$.ready_fn()
      while (private$.running) Sys.sleep(0.1)
    }
  ),

  private = list(
    .title    = NULL, .width    = NULL, .height   = NULL,
    .icon     = NULL, .ready_fn = NULL, .running  = FALSE,
    .handlers = NULL, .ws_session = NULL, .server = NULL,

    find_free_port = function() {
      # httpuv trick: bind port 0, get assigned port, release
      con <- serverSocket(0)
      port <- con$getsockname()$port
      con$destroy()
      port
    },

    start_server = function(port) {
      self_ref <- self
      private$.server <- httpuv::startServer("127.0.0.1", port, list(
        onWSOpen = function(ws) {
          private$.ws_session <- ws
          ws$onMessage(function(binary, msg) {
            tryCatch({
              parsed <- jsonlite::fromJSON(msg, simplifyVector=FALSE)
              handler <- private$.handlers[[parsed$type]]
              if (!is.null(handler)) handler(parsed$payload)
            }, error = function(e) message("[RDesk] handler error: ", e$message))
          })
        },
        call = function(req) {
          # Serve static files from www/
          list(status=200L, headers=list("Content-Type"="text/html"),
               body="<h1>RDesk server running</h1>")
        }
      ))
    }
  )
)

12.1  Code quality rules
Every exported R function must have a roxygen2 comment block (@title, @param, @return, @export)
Every R file must have a file-level comment at line 1 describing its purpose
C++ code must compile cleanly with -Wall at CXX17 standard — zero warnings
No hardcoded port numbers anywhere — always use the find_free_port() helper
Never call Sys.sleep() for more than 0.5 seconds in the main run loop — use httpuv event polling
12.2  Architecture rules
The webview window must run on the main thread — never launch it in a background thread (macOS will crash)
httpuv server must start before the window opens — the window navigates to the already-running server
All R ↔ UI messages must go through the WebSocket channel — never use shared global state
The App R6 class must be the only public interface — all httpuv and C++ calls are private
12.3  Process rules
Complete Phase 0 fully before starting Phase 1 — do not mix phases
After each phase, run R CMD check and fix all ERRORs before continuing
After Phase 1, manually test window opens on the target OS before proceeding
